## Spacing Scale

Declared values (multiples of 4). Derived from existing style.css measurements, stated in px for precision.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Table cell padding (4px 6px), icon/inline gaps |
| sm | 8px | Compact vertical padding, pill button vertical padding (0.3rem ≈ 5px — round to 8px for new elements) |
| md | 16px | Default horizontal element spacing, section padding (gap: 1rem on nav) |
| lg | 24px | Section/content padding (methodology padding: 1.5rem top) |
| xl | 32px | Layout gaps where needed |
| 2xl | 48px | Major section breaks |

- Toolbar padding: 0.6rem 1rem (≈ 10px 16px) — matches existing; use exact values when extending toolbar

- Toolbar separator: 1px wide, 24px tall — match on top.html if toolbar pattern used
- Score badge / pillar sub-score chips / trap badge: padding 4px 8px, border-radius 4px (on the 4px grid)

### OverallScore color thresholds

Apply OverallScore coloring inline via JS (same pattern as makeSignalFormatter):
- Score >= 70: green (`#a3be8c` / `#2e3440`)
- Score 40–69: yellow (`#ebcb8b` / `#2e3440`)
- Score < 40: red (`#bf616a` / `#eceff4`)

These thresholds match the intent from CONTEXT.md that high scores read as good and traps stand out.

### top.html ranked card component

Each stock in the Top N is rendered as a card. Cards are stacked vertically in a single column (not a grid) for clear rank ordering.

```
┌─────────────────────────────────────────────────────────────────┐
│  #1  TICKER  $Price    [OVERALL: 87] [green]      [TRAP] ← only │
│                                                      if tripped  │
│  Value: 72  Quality: 65  Growth: 58  Safety: 91                 │
│  Lynch: Buy   Graham: Deep Buy   Defensive: Pass                │
└─────────────────────────────────────────────────────────────────┘
```

Card layout specification:
- **Row 1 (identity + score):** Rank number (#N), Ticker (monospace, link to Finviz matching dashboard pattern), Price (2 decimal places), OverallScore badge (color-coded green/yellow/red per threshold above), value-trap badge if `is_trap = true`
- **Row 2 (pillar sub-scores):** Four labeled chips: `Value: N  Quality: N  Growth: N  Safety: N` — neutral chip styling (no color coding on pillar chips themselves; the OverallScore badge carries the summary color signal)
- **Row 3 (headline signals):** Lynch status, Graham status, Defensive label — text only, color-coded background matching SIGNAL_COLORS exactly (same green/yellow/red as dashboard)
- Card background: `#3b4252` (Polar Night 1 — same as table row)
- Card border: `1px solid #4c566a`
- Card border-radius: 6px
- Card padding: 12px 16px (sm + md)
- Card margin-bottom: 8px (sm)
- Ticker link: `color: inherit; text-decoration: underline` — matches dashboard Finviz link pattern
- Max-width of card list: 860px, centered (matches `.methodology-content` max-width)
- Padding of list container: 1.2rem (matches methodology)

## Interaction Contract

### 10/25 count toggle

- Default state: Top 10 active (btn-top-10 has class `active`, `aria-pressed="true"`)
- On click Top 25: deactivate Top 10, activate Top 25, re-slice and re-render the card list (client-side, no refetch)
- On click Top 10: deactivate Top 25, activate Top 10, re-slice and re-render
- Toggle is mutually exclusive (radio-group behavior, implemented via pill buttons)
- No URL state persistence required in Phase 5

### Data fetch

- Fetch: `data/results.json?v=` + `Date.now()` (cache-busting, identical to index.html)
- Sort: sort rows by `OverallScore` descending before slicing (client-side)
- Slice: top N rows after sort (N = 10 or 25 per toggle state)
- Filter: exclude rows where `Error` is non-null (same rule as index.html `noErrorFilter`)
- On load: call `updateFreshnessUI(data.generated_at)` and `buildNav("top")`
- On fetch error: set freshness badge to `Error loading data`, render empty state copy in `#top-container`
- On empty result set after filtering: render empty state copy in `#top-container`

### app.js loading

- `<script src="app.js"></script>` loads before page-specific script in both index.html and top.html
- methodology.html only needs `buildNav("methodology")` — load app.js there too for nav consistency
- app.js exposes all shared functions as module-level variables (no ES modules — vanilla `var`/`function` declarations, matching existing index.html convention)

### Accessibility

- `nav.main-nav` has `aria-label="Site navigation"` (existing pattern — apply on top.html)
- Active nav link has `aria-current="page"` (existing pattern)
- Count toggle buttons have `aria-pressed` state updated on each click
- Stale banner is always in DOM, toggled via `.visible` class (existing pattern — no `aria-live` needed; it's a structural warning, not a dynamic alert)
- Card list: no ARIA list role required (div-based layout is acceptable for this tool)
- Trap badge: no additional ARIA — the text "TRAP" is sufficient for screen readers at this tool's scope
