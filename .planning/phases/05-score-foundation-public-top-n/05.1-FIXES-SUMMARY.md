## Bug A — value-trap gate tripped for 100% of tickers (dead Safety pillar)

**Symptom (live data):** every row had `is_trap: true`, `score_safety: 0.0`, `EPS_Stability: 0.0`.

**Root cause:** `trap_gate()` consumed the defensive `EPS_Stability` flag, which is
`int(positive EPS in ≥ 8 of last 10 years)`. yfinance `income_stmt` returns only ~4
years of annual EPS, so `pos_eps_yrs (≤4) >= 8` is never true → `EPS_Stability = 0` for
every ticker → the gate's `eps_stability == 0` check tripped universally → Safety floored
to 0 for the whole universe (D-03). A trap flag that is always on carries no signal.

**Fix:** new pure helper `_eps_stable_for_gate(annual_eps)` judges stability over the
*available* window — stable (1) only if every available year had EPS > 0, unstable (0) if
any year ≤ 0, None when < 2 years (D-01b unknown). The trap call site now uses it instead
of the 8-of-10 defensive flag. Healthy all-positive-EPS stocks no longer trip; loss-year
names (e.g. ALL, CINF) still do.

> The defensive `EPS_Stability` criterion itself (Graham 8-of-10) is left unchanged — it
> under-scores every row by 1 point uniformly given 4-yr data, which is a uniform shift,
> not a ranking distortion. Longer EPS history is a Phase 6/7 data concern.

## Bug B — growth was inflated Finnhub `epsGrowth5Y` capped at 25% for ~every ticker

**Symptom (live data):** nearly every row showed `Growth_g_Pct: 25.0` (= `GROWTH_CAP`) and
`Lynch_Category: "Fast"`, including flat/declining-EPS names (CTSH `[4.42,4.21,4.52,4.57]`,
HBAN `[1.47,1.26,1.24,1.41]`, LEN `[15.74,13.73,14.31,7.98]`). This inflated Lynch/Graham
fair values → fake 40–70% discounts → maxed the Growth pillar → the Top-N became a
"Finnhub-over-reported-growth" list rather than a value list.

**Root cause:** `g = fund["growth_pct"]` (Finnhub) was trusted, then `min(g, GROWTH_CAP)`.
The cap only guards the top tail; garbage capped at 25% is still a fake 25% grower. The
realized EPS CAGR was only used as a fallback when Finnhub was absent.

**Fix:** new pure helper `_reconcile_growth(g_finnhub, g_cagr)` anchors reported growth to
the realized CAGR — takes the lower value when both exist, falls back to whichever is
present. Effect on the example tickers: CTSH → ~1% (not 25%); HBAN/LEN → negative →
route to `WORST_DISCOUNT` (drop to bottom); genuine growers (e.g. L ~33%) stay at the cap.

> ponytail: realized CAGR is endpoint-sensitive (first/last EPS in the window) — good
> enough as an interim anchor; Phase 7 refines growth handling.
