<objective>
Close the three verified, reproducible defects from 07-VERIFICATION.md (mirroring 07-REVIEW.md CR-01/CR-02/CR-03). This is a gap-closure plan — it fixes exactly these three defects and adds regression coverage for the two that are unit-testable. It does NOT replan or re-touch any correctly-implemented Phase 7 work.

Gap 1 (CR-02): Piotroski F5 has an inverted fail-safe — a missing current-year long-term-debt defaults the ratio to 0, which makes the "leverage decreased" `<` comparison almost always True, silently awarding the point. This inflates the F-Score and therefore the Safety pillar on missing data.

Gap 2 (CR-03): The reconciled growth rate `g` fed into the DCF helpers is only upper-capped (`min(g, GROWTH_CAP)`), never floored. A severely negative Finnhub `epsGrowth5Y` (below -100%) makes `(1+g)` negative, producing sign-oscillating/negative projected EPS and a nonsensical intrinsic value that scores as a false "deep value" BUY. Additionally, the D-10 `[CYCLICAL]` coverage flag for Energy/Basic Materials was never implemented.

Gap 3 (CR-01): The "Commit monthly snapshot" step in screener.yml runs `python -c "import stock_screener; ..."` with no `env:` block. Because stock_screener reads `os.environ["FRED_API_KEY"]`/`["FINNHUB_API_KEY"]` at import time, this raises `KeyError` in CI every first-weekday-of-month, so snapshots never land.

Purpose: Make the Safety pillar, DCF signal, and monthly snapshot feature behave as Phase 7 designed before the phase is marked complete.
Output: Corrected stock_screener.py (F5 guard, growth floor, cyclical flag), corrected screener.yml, and two new regression tests.
</objective>
